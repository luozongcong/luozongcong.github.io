查找了一下，可以操作excel表的几个库有以下几个：

  • openpyxl
    这个是推荐使用的库，可以读写Excel 2010以上格式，以.xlsx结尾的文件。

  • xlsxwriter
    这个支持.xlsx,但是只支持写入，格式化等操作，不支持读取。

  • xlwt
    这个和上面的相对应，支持写入书和格式化数据，支持xls结尾的文件格式。

    这个是整合了xlrd和xlwt两个库的功能。

Winows平台Python2与Python3共存

发表于 2016-05-07   |   分类于 技术

关于Windows平台上Python2与Python3共存，使用过程中可能会遇到一些问题。
下面是我从网上搜集过来的正确使用方法，可以完美使用。具体也可以参考官方说明3.4小节.
在安装Python3时按照下图，勾选Install python launcher for all users.

关于调用不同版本Python

如果同时安装了Python2和Python3，那么在使用的时候可以这样：

  • 使用py -2 hello.py即可调用Python2执行hello.py。py -3 hello.py即可调用Python3来执行hello.py。
  • 在hello.py文件的开头进行标记（可能叫法不对）在文件第一行写上#！python2，或者#!python3这样标记以后，在执行的时候只需要输入py hello.py,会自动调用相应的解释器来执行文件。
    记得#!python2一定要放在第一行。

关于pip的使用

  • 若需要安装Python2的库，执行py -2 -m pip install xxxx即可，xxxx是库的名字。
    若需要安装Python3的库，执行py -3 -m pip install xxxx即可。

第二周总结

发表于 2016-04-01   |   分类于 随笔

第二周了,简单回顾一下本周学习的知识，也算对自己成长路径的记录吧。
上周初步接触了Django，了解了一些简单页面的逻辑。这周开始就接触了AngularJS,由于对javaScripts是0基础，所以一开始简直是懵了。这都是神马东西。硬着头皮看了一个AngularJS和Django配合所做的小项目，结果肯定是没看懂了。
第二天在同事的指点下，对AngularJS的基本用法进行了了解，对基本的一个控制模式和常用的指令进行掌握，但是，这只是概念性的了解，然后我要做的是模仿之前的项目自己写出来一个作为对新知识的练习。可想而知。。

最后是在同事的指点下，一步一步给写出来了，虽然很简单，但是对AngularJS的认识确实加深了许多。

周四的时候，在目前公司的项目基础上添加新的功能，所以先写了两个静态页面，并通过AngularJS进行跳转交互，之后通过视图函数和数据库连接获取数据再通过AngularJS传递给静态页面，实现了用户token生产，帐号，密码写入等操作。

开始，确实挺痛苦的，要补的太多，对于网络之间的请求，数据传递模模糊糊的，通过实际的编码，确实有了更深入的了解。

一周总结

发表于 2016-03-26   |   分类于 随笔

上班一周时间了，前两天就是看Django文档了，之前有Flask的基础在，看起来还是比较轻松的。反而感觉Django刚开始需要配置的东西很少，觉得比Flask好用。实际上，web框架的基本架构是一致的，都是MVC架构。所以看起来也是比较有层次。毕竟之前折腾过Flask，现在Django配置起来还算得心应手。官方文档上的投票小应用比较简单，只是对整体框架的一个大概认识，对整体有个大致的了解，接下来还需要对每部分进行深入学习。

后两天，就开始着手干一点实际的活了。公司之前有一个基于Flask的小项目，现在想统一下技术栈，正好交给我，让我把其迁移到Django上。初步看了一下，视图层还是比较好做，把原来的Html模版拿过来改改就可以直接用了，把视图函数重新写写，就可以实现页面逻辑了。

接下来的一段时间，重心还是在Django和项目的API上，要重点把Django中的各种实现方式理解下，了解API的设计模式和思路。先写这么多吧。
